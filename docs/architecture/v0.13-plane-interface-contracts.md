# v0.13 Plane Interface Contracts

## Purpose

This document defines how PostgreSQL, TigerBeetle, pgvector, DuckDB, formula workers, and live-delivery components interact without creating hidden write paths or conflicting sources of truth.

## Universal envelope

Any data crossing a plane boundary must carry, where applicable:

```text
tenant_id
object_type
object_id
source_version
command_id
trace_id
correlation_id
source_high_watermark_outbox_id
schema_hash
permission_scope_hash
data_classification
created_at
```

## Allowed flows

| Flow | Allowed? | Contract |
|---|---:|---|
| API -> PostgreSQL command handler | Yes | command_log and domain transaction. |
| API -> TigerBeetle direct | No | Must go through NumericLedgerPort after domain validation. |
| API -> DuckDB direct | No | Analytics jobs are asynchronous and derived. |
| API -> pgvector direct | Limited | Retrieval API only, with permission filter. |
| PostgreSQL -> TigerBeetle | Yes | Deterministic movement plan through NumericLedgerPort. |
| TigerBeetle -> PostgreSQL | Yes | Projection repair, reconciliation, transfer status. |
| PostgreSQL -> pgvector | Yes | Derived chunks/embeddings only. |
| PostgreSQL -> DuckDB | Yes | Governed snapshots, Parquet artifacts, read-only internal bridge. |
| pgvector -> DuckDB | Indirect | Candidate object IDs must pass deterministic permission filters. |
| DuckDB -> PostgreSQL command mutation | No | Results may create suggestions; user actions still submit commands. |

## Failure isolation

| Failed plane | Required behavior |
|---|---|
| TigerBeetle unavailable before cutover | Use PostgreSQL MVP adapter if ledger not cut over; otherwise fail closed and queue repair. |
| pgvector unavailable | Fall back to deterministic search/filter; disable semantic assistant features. |
| DuckDB unavailable | Disable analytics/export jobs; operational edits continue. |
| Formula workers unavailable | Use stale-safe display or scoped formula disablement. |
| SSE unavailable | Client can poll/refresh from PostgreSQL projections. |

## Mutation rule

```text
No specialized plane may cause source-of-truth mutation except by returning to a command handler.
```

## Query-answering rule

AI or analytics answers that cross planes must cite deterministic records from PostgreSQL projections, TigerBeetle-derived ledger projections, or DuckDB result artifacts. Similarity score alone is never evidence.

## pgvector strategy refinement

The pgvector plane uses the selected model from `docs/data/pgvector-integration-strategy-options.md`:

```text
permissioned derived chunk registry
+ hybrid lexical/vector retrieval
+ evidence-gated ANN indexes
+ optional dedicated semantic PostgreSQL/pgvector database after scale evidence
```

This means pgvector is not a broad AI query router and not an authorization or mutation plane. It narrows context; deterministic PostgreSQL, TigerBeetle-derived projections, and DuckDB analytics provide facts. User-visible retrieval requires permission, classification, and source-version revalidation.


## Outbox plane interface

| Field | Requirement |
|---|---|
| Input | PostgreSQL transaction inserts `outbox_events` with `event_id`, `idempotency_key`, `route_key`, `partition_key`, `target_planes`, `payload_hash`, tenant and classification fields. |
| Output | SSE envelopes, derived-plane jobs, CDC/broker events, or integration events. |
| Authority | PostgreSQL `outbox_events` until a future ADR changes it. Broker offsets and CDC LSNs are delivery state only. |
| Failure mode | Consumer lag marks derived planes stale; command commits continue. Replay uses outbox high watermark and consumer checkpoints. |
| Prohibited | Direct operational mutation from broker event; CDC inference from arbitrary operational table changes; broker publish in command handler instead of outbox insert. |

## v0.14 external integration extension

External systems are added as a post-MVP connector plane. They do not become a source of operational mutation authority.

```text
Inbound external data -> integration intake/staging -> command proposal/submission -> command handler
Outbound external data -> committed outbox envelope -> integration dispatcher -> external system
```

This preserves PostgreSQL command/outbox authority while allowing later integrations with SaaS APIs, webhooks, SCIM, file/batch exchange, DuckDB snapshots, pgvector content feeds, and TigerBeetle reconciliation workflows.
